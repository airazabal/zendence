## Build & Development Commands

```bash
./gradlew assembleDebug          # Build debug APK
./gradlew assembleRelease        # Build release APK
./gradlew installDebug           # Build and install on connected device
./gradlew test                   # Run unit tests
./gradlew connectedAndroidTest   # Run instrumented tests (requires device/emulator)
./gradlew lint                   # Run lint checks
./gradlew clean                  # Clean build outputs
```

Requires Android Studio Ladybug or newer. Min SDK 26 (Android 8.0), target/compile SDK 35.

## Architecture

Single-module Android app using **MVVM + Repository** pattern with 100% Jetpack Compose UI:

- **`MainActivity.kt`** — Single activity; hosts all Compose UI, observes `MeditationViewModel`
- **`MeditationViewModel.kt`** — `AndroidViewModel`; owns all UI state and orchestrates business logic
- **`MeditationRepository.kt`** — Data layer; abstracts Room DB and SharedPreferences, exposes Kotlin Flow APIs
- **`Database.kt`** — Room database definition, DAOs, and entity classes
- **`Models.kt`** — Shared data classes used across layers
- **`MeditationService.kt`** — Foreground `MediaSessionService` (Media3/ExoPlayer) for background audio playback
- **`AiIntelligence.kt`** — Gemini SDK integration for AI-driven session analysis and daily wisdom
- **`DailyReminderWorker.kt`** — WorkManager worker for scheduled daily reminder notifications
- **`MeditationContentProvider.kt`** — Content provider that exposes saved session data to other apps
- **`ZendenceApp.kt`** — Application class for app-level initialization

UI components live in `ui/components/` (7 Compose composables for timer, presets, history, settings, AI trends, dialogs). Theme files are in `ui/theme/`.

## Key Technical Details

**Room**: Entities and DAOs are defined in `Database.kt`. The KSP annotation processor generates Room implementation code — run a build after schema changes.

**Media3/ExoPlayer**: `MeditationService` extends `MediaSessionService`. Audio assets are bundled in `res/raw/`.

**Gemini AI**: Uses `com.google.ai.client.generativeai` SDK. The API key is managed via `BuildConfig` (set in local `gradle.properties`, not committed).

**Version management**: `version.properties` auto-increments `VERSION_CODE` and `VERSION_NAME` on each build — do not edit this file manually.

**Serialization**: Uses `kotlinx.serialization` (not Gson/Moshi) for JSON handling.

**Content Provider authority**: `com.alex.zendence.provider` — used by `MeditationContentProvider` to share session history with external apps.
